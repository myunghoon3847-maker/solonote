# 훈노트 v4.5.3 변경 기록

## 사용자 데이터 분리 보강

- `loadMemosFromCloud()` 조회에 현재 로그인 사용자의 `user_id` 필터를 추가했습니다.
- 레거시 `프로젝트` 카테고리를 `업무`로 바꾸는 쿼리에도 같은 사용자 필터를 추가했습니다.
- Supabase RLS가 최종 보안 경계라는 원칙은 유지하며, 클라이언트 쿼리에도 소유자 조건을 명시했습니다.

## 백업 복원 보안

- 복원 전 파일 크기, 메모·카테고리·체크리스트 개수와 필드 형식을 검사합니다.
- 제목·내용·프로젝트·카테고리·체크리스트 길이를 제한합니다.
- null 문자가 포함된 잘못된 텍스트와 유효하지 않은 날짜 형식을 거부합니다.
- 악성 또는 손상된 체크리스트 ID를 안전한 ID로 교체하고 중복 ID를 정리합니다.
- HTML 문자열에 들어가는 메모·체크리스트 ID를 이스케이프합니다.

## 입력·버전 관리

- 제목 300자, 내용 200,000자, 프로젝트 100자, 체크리스트 항목 500자 제한을 적용했습니다.
- 체크리스트는 메모 하나에 200개까지 추가할 수 있습니다.
- 저장 직전에도 같은 제한을 검사해 HTML 속성 우회를 방지합니다.
- 백업 형식과 계정 삭제 요청의 클라이언트 버전을 `4.5.3`으로 통일했습니다.
- 화면 표시, 정적 자원 쿼리, 서비스워커 캐시를 `453`으로 갱신했습니다.

## 변경하지 않은 항목

- Supabase 테이블·RLS·함수는 변경하지 않았습니다.
- 기존 메모·카테고리·체크리스트의 의미와 ID는 유지합니다. 안전하지 않거나 중복된 체크리스트 ID만 앱에서 정규화합니다.
- UI 배치, 색상, Android 패키지 설정은 변경하지 않았습니다.
