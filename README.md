# 훈노트 v4.5.2

훈노트(HoonNote) v4.5.2는 메모 카테고리와 독립 `할 일` 화면의 역할을 분리하고, 삭제할 수 없는 보호 카테고리를 색상으로 구분한 정리 버전입니다.

## v4.5.2 수정 내용

- 상단 `메모·할 일` 화면 탭은 유지
- 메모 작성 선택창과 카테고리 필터에서 `할 일` 제거
- 기존 `할 일` 카테고리 메모는 `미분류`로 호환해 누락 방지
- `중요·보관·미분류`를 연한 스카이블루 보호 카테고리로 표시
- 보호 카테고리 선택 시 진한 스카이블루로 상태 표시
- CSS·JavaScript·서비스 워커 버전을 `v=452`로 일괄 갱신

## 유지되는 v4.5 기능

- 사용자별 카테고리를 Supabase에 저장하고 여러 기기에서 동기화
- `카테고리 관리` 창에서 새 카테고리 추가
- 기존 카테고리 이름 변경 시 연결된 모든 메모도 함께 변경
- 카테고리 삭제 시 메모를 삭제하지 않고 `미분류`로 이동
- 같은 이름·빈 이름·20자 초과·시스템 이름 등록 방지
- 카테고리를 최소 1개 유지
- JSON 백업에 카테고리 목록 포함
- v4.4 이전 백업과 기존 메모 호환

## 카테고리 구분

- 사용자 카테고리: `업무·아이디어·일상` 및 사용자가 추가한 카테고리
- 보호 카테고리: `중요·보관·미분류` — 추가·이름 변경·삭제 불가
- 화면 기능: 상단 `할 일` 탭과 `휴지통`은 메모 카테고리가 아닌 별도 화면

## 배포 순서

1. 현재 운영 데이터를 JSON으로 백업합니다.
2. 이 폴더의 웹 파일을 일부가 아닌 전체 교체 방식으로 GitHub Pages 저장소에 배포합니다.
3. `https://myunghoon3847-maker.github.io/solonote/?v=452`에서 v4.5.2 표시를 확인합니다.
4. 설치 앱에서 업데이트 안내가 보이면 `업데이트`를 누릅니다.
5. `TEST_CHECKLIST_v4.5.2.md`에 따라 PC와 스마트폰에서 검수합니다.

v4.5의 `05_create_memo_categories.sql`을 이미 실행했다면 추가 SQL은 없습니다. 이번 변경은 웹앱 UI와 호환 처리이며 기존 TWA 래퍼 설정을 바꾸지 않으므로 새 AAB는 필요하지 않습니다.
