# 훈노트 v4.5.5

훈노트(HoonNote) v4.5.5는 홈 이동, 모바일 입력 안정화, 휴지통 화면 단순화, 로고 정리와 작성 화면의 카테고리 관리를 반영한 UI·UX 수정 버전입니다.

## v4.5.5 수정 내용

- 왼쪽 상단 훈노트 로고를 홈 버튼으로 변경
- 로고를 파란 `H` 심볼과 간결한 워드마크 조합으로 정리
- 로고 클릭 시 메모 홈으로 이동하고 검색·카테고리·정렬 필터를 기본값으로 초기화
- 작성 중인 내용이 있으면 로고 이동 시 삭제하지 않고 로컬 초안으로 보존
- 휴지통의 `메모로 돌아가기` 버튼 삭제
- 스마트폰 확대·축소를 제한하는 viewport 설정 적용
- 모바일 입력창 글자 크기를 16px로 조정해 iPhone 등의 입력 자동 확대 방지
- 메모 작성 화면 카테고리 항목에 `추가·삭제` 버튼 추가
- 기존 카테고리 관리 모달과 연결하여 추가·이름 변경·삭제 기능 재사용
- PWA 캐시·화면·백업·계정 삭제 요청 버전을 `4.5.5`로 갱신

## 주소창 또는 링크 표시 안내

웹브라우저에서 일반 웹페이지로 접속한 경우 브라우저 주소창은 웹 코드로 강제로 없앨 수 없습니다. 홈 화면에 설치한 PWA는 `standalone` 모드로 실행되며 주소창이 표시되지 않습니다.

Google Play의 TWA 앱 위에 사이트 주소가 표시된다면 Digital Asset Links 인증을 완료해야 합니다. 자세한 절차는 `ANDROID_ADDRESS_BAR_FIX_v4.5.5.md`를 확인하세요. 실제 Play 앱 서명 SHA-256 값은 이 배포본에 포함되지 않습니다.

## 유지되는 기능

- 로그인·회원가입·비밀번호 재설정·계정 삭제
- 사용자별 Supabase 클라우드 저장과 여러 기기 동기화
- 메모 작성·수정·검색·중요 표시·보관·휴지통·영구 삭제
- 체크리스트와 할 일 화면
- 사용자 카테고리 추가·이름 변경·삭제
- JSON 백업·추가 복원과 이전 백업 호환
- PWA 설치·업데이트와 Android TWA

## 배포 순서

1. 운영 데이터를 JSON으로 백업합니다.
2. 이 폴더 내부 파일 전체를 GitHub Pages 저장소의 기존 `/solonote/` 파일과 교체합니다.
3. 배포 후 `https://myunghoon3847-maker.github.io/solonote/?v=455`에 접속합니다.
4. 메뉴 하단에 `v4.5.5`가 표시되는지 확인합니다.
5. 설치 앱에 업데이트 안내가 나타나면 적용합니다.
6. `TEST_CHECKLIST_v4.5.5.md`에 따라 스마트폰과 PC에서 확인합니다.

이번 버전은 데이터베이스 구조를 변경하지 않으므로 추가 Supabase SQL은 필요하지 않습니다.

## 주요 문서

- `CHANGES_v4.5.5.md`: 변경 기록
- `TEST_CHECKLIST_v4.5.5.md`: 실기기 검수표
- `RELEASE_CHECKLIST_v4.5.5.md`: 배포·롤백 절차
- `VALIDATION_REPORT_v4.5.5.md`: 자동 검수 결과
- `ANDROID_ADDRESS_BAR_FIX_v4.5.5.md`: Android 주소 표시 제거 절차

## Android 기준값

- 표시 이름: 훈노트
- 패키지명: `com.hooncompany.hoonnote`
- target API: `36`
- 배포 형식: `AAB`
- 앱 서명: Google Play App Signing

실제 AAB, APK, Play 앱 서명 SHA-256과 서명키는 이 ZIP에 포함되어 있지 않습니다.
