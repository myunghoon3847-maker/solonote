# SoloNote v1

1인 업무노트 / 아이디어노트 웹앱 1차 버전입니다.

## 실행 방법

1. 압축을 풉니다.
2. `index.html` 파일을 더블 클릭합니다.
3. 브라우저에서 메모를 작성하고 저장합니다.

## 포함 기능

- 메모 작성
- 메모 저장
- 메모 목록 보기
- 메모 상세 보기
- 메모 수정
- 메모 삭제
- 카테고리 필터
- 검색
- 브라우저 localStorage 저장
- 스마트폰 화면 대응

## 파일 구조

```text
solonote_v1/
├─ index.html
├─ css/
│  └─ style.css
└─ js/
   ├─ storage.js
   ├─ ui.js
   └─ app.js
```

## 주의

현재 버전은 서버 없이 브라우저에 저장됩니다.
브라우저 캐시/사이트 데이터를 삭제하면 메모가 사라질 수 있습니다.


## v1.1 수정 사항

- 스마트폰 화면에서 상단 헤더 오른쪽 간격이 맞지 않는 문제 수정
- 모바일 가로 넘침 방지
- 헤더, 패널, 레이아웃 폭을 100% 기준으로 정리


## v1.2 수정 사항

- 모바일 기준 CSS 구조로 재정리
- 오른쪽으로 화면이 밀리거나 입력창이 잘리는 문제 방지
- 상단 `0개의 메모` 박스를 작은 배지 형태로 수정
- 모바일에서 헤더, 입력폼, 패널 폭을 더 안정적으로 정리
- 긴 한글/영문 입력 시 레이아웃이 깨지지 않도록 줄바꿈 보강


## v1.3 수정 사항

- 카테고리 탭에 `휴지통` 추가
- 삭제하기 동작 변경: 완전 삭제가 아니라 휴지통으로 이동
- 휴지통에서 `복구하기` 가능
- 휴지통에서 `완전 삭제` 가능
- 기존 v1/v1.2 메모 데이터와 호환되도록 `isDeleted` 기본값 보정
- 상단 메모 개수는 휴지통을 제외한 일반 메모 기준으로 표시


## v1.4 수정 사항

- 메모 작성 폼에 `중요 메모로 표시` 체크박스 추가
- 카테고리 탭에 `중요` 추가
- 중요 메모는 목록에서 별표와 중요 배지 표시
- 상세 보기에서 `★ 중요 · 카테고리` 형태로 표시
- 작성/수정 시 `isImportant` 값 저장
- 기존 v1.3 이하 메모 데이터와 호환되도록 `isImportant` 기본값 보정


## v1.5 수정 사항

- 메모 작성 영역 접기/펼치기 기능 추가
- 앱 첫 접속 시 작성 폼은 접힌 상태로 시작
- `+ 새 메모 작성` 버튼을 누르면 작성 폼 펼침
- `작성 영역 닫기` 버튼으로 작성 폼 접기
- 저장 완료 후 작성 폼 자동 접기
- 수정하기 클릭 시 작성 폼 자동 펼침
- 수정 취소 시 작성 폼 자동 접기
- 스마트폰에서 메모 목록 접근성 개선


## v1.6 수정 사항

- 백업하기 기능 추가
- 현재 저장된 메모 전체를 JSON 파일로 다운로드
- 복원하기 기능 추가
- JSON 백업 파일을 선택해 기존 메모에 합치기 방식으로 복원
- 같은 id의 메모는 중복 복원하지 않음
- 중요 메모, 휴지통 상태, 작성일/수정일 정보까지 백업/복원
- 서버와 로그인 없이 localStorage 데이터 손실 위험을 줄이는 기능 추가


## v1.6.1 수정 사항

- 백업/복원 버튼 간격과 정렬 수정
- 모바일에서 백업/복원 버튼이 깔끔하게 2열로 보이도록 수정
- 파일 선택 input이 화면에 노출되는 문제 수정
- 복원하기 버튼을 누를 때만 파일 선택창이 뜨도록 수정


## v1.6.2 수정 사항

- 파일 선택 input을 HTML에서 완전히 제거
- 복원하기 버튼 클릭 시에만 자바스크립트로 임시 파일 선택창 생성
- 화면에 `파일 선택 / 선택된 파일 없음` UI가 노출되는 문제 해결
- 백업/복원 버튼을 2열 버튼 형태로 정리

- 홈화면 추가용 앱 아이콘 추가
- 스마트폰에서 홈화면에 추가해 앱처럼 실행 가능
- GitHub Pages 환경을 고려해 상대 경로 기준으로 PWA 파일 연결


## v1.8 수정 사항

- 메모 정렬 기능 추가
- 최근 수정순 정렬
- 최신 작성순 정렬

- 메모 안 체크리스트 기능 추가
- 새 메모 작성 시 할 일 항목 추가 가능
- 작성 중인 체크리스트 항목 삭제 가능
