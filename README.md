# 훈노트 v4.5.4

훈노트(HoonNote) v4.5.4는 새 기능을 늘리지 않고, 개인용 메모 앱으로 더 간단하고 편하게 사용할 수 있도록 기존 화면과 뒤로가기를 정리한 UI·UX 안정화 버전입니다.

## v4.5.4 수정 내용

- 프로젝트 입력·빠른 선택·필터·검색·카드·상세 표시를 화면에서 제거
- 기존 메모의 프로젝트 값과 백업 호환 데이터는 삭제하지 않고 보존
- 필터 버튼과 접힌 필터 패널 제거
- 카테고리 UI는 유지하고 정렬 5종을 작은 연한 텍스트로 메인 화면에 표시
- 정렬 항목 사이에 `ㅣ` 구분선을 사용하고 작은 화면에서는 가로 스크롤 없이 줄바꿈
- 메인 화면의 메모 개수 표시와 빈 화면의 `전체 메모 보기` 버튼 제거
- 메뉴 상단 휴지통 버튼을 삭제하고 데이터 관리의 휴지통 표시를 실제 버튼으로 변경
- 메모 내용 입력 영역의 기본 세로 높이 확대
- 스마트폰·PWA 뒤로가기로 메뉴, 작성창, 상세, 카테고리 관리, 계정 삭제를 먼저 닫도록 개선
- 할 일·휴지통 화면에서 뒤로가면 이전 앱 화면으로 복귀하도록 브라우저 기록 연결
- 화면·백업·계정 삭제 요청·PWA 캐시 버전을 `4.5.4`로 통일

## 유지되는 기능

- 로그인·회원가입·비밀번호 재설정·계정 삭제
- 사용자별 Supabase 클라우드 저장과 여러 기기 동기화
- 메모 작성·수정·검색·중요 표시·보관·휴지통·영구 삭제
- 체크리스트와 `할 일` 화면
- 사용자 카테고리 추가·이름 변경·삭제
- JSON 백업·추가 복원과 이전 백업 호환
- PWA 설치·업데이트와 Android TWA
- v4.5.3의 백업 검증·XSS 방어·사용자별 쿼리 조건

## 배포 순서

1. 운영 데이터를 JSON으로 백업합니다.
2. 이 폴더의 파일 전체를 GitHub Pages 저장소의 기존 `/solonote/` 파일과 교체합니다.
3. `https://myunghoon3847-maker.github.io/solonote/?v=454`에 접속합니다.
4. 메뉴 하단의 `v4.5.4` 표시를 확인합니다.
5. 설치 앱에 업데이트 안내가 보이면 적용합니다.
6. `TEST_CHECKLIST_v4.5.4.md`에 따라 스마트폰과 PC에서 확인합니다.

이번 버전은 데이터베이스 구조를 바꾸지 않으므로 추가 SQL과 새 AAB가 필요하지 않습니다. 기존 프로젝트 값도 DB에서 삭제하지 않습니다.
