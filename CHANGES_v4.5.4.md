# 훈노트 v4.5.4 변경 기록

## 개인용 앱 방향에 맞춘 단순화

- 프로젝트 기능을 작성 화면, 빠른 선택, 필터, 검색 결과, 메모 카드, 상세 화면과 할 일 화면에서 제거했습니다.
- 기존 메모의 `project` 값은 DB·백업 호환을 위해 유지합니다.
- 새 메모에는 빈 프로젝트 값이 저장되며, 기존 메모를 수정해도 숨겨진 프로젝트 값이 임의로 삭제되지 않습니다.

## 탐색 화면

- 필터 버튼과 접힌 필터 패널을 제거했습니다.
- 기존 카테고리 버튼 디자인과 배치는 유지했습니다.
- 정렬 5종을 카테고리 아래에 작은 연한 텍스트로 표시했습니다.
- 항목 사이는 `ㅣ` 구분선을 사용하며, 화면이 좁으면 가로 스크롤 없이 다음 줄로 이어집니다.
- 헤더와 메모 목록 위의 메모 개수 표시를 제거했습니다.
- 빈 화면의 `전체 메모 보기` 버튼과 프로젝트 안내 문구를 제거했습니다.

## 작성·데이터 관리

- 메모 본문 입력 영역의 기본 높이를 확대했습니다.
- 메뉴 상단의 별도 휴지통 영역을 제거했습니다.
- 데이터 관리 통계 안의 `휴지통`을 실제 버튼으로 바꾸고 삭제된 메모 수를 함께 표시합니다.

## 뒤로가기

- 브라우저·Android·PWA 뒤로가기를 앱 내부 기록과 연결했습니다.
- 메뉴, 작성창, 상세 보기, 카테고리 관리, 계정 삭제 창이 열려 있으면 해당 화면을 먼저 닫습니다.
- 작성 중인 내용이 있으면 뒤로가기 전에 확인하고 자동 저장 초안을 유지합니다.
- 할 일·휴지통 같은 내부 화면은 뒤로가기로 이전 앱 화면에 복귀합니다.
- 메모 메인 화면에서만 이전 웹페이지 이동 또는 앱 종료가 허용됩니다.

## 보안·호환성

- v4.5.3의 백업 스키마·크기 제한, 체크리스트 ID 정규화, HTML 이스케이프와 사용자별 클라우드 쿼리를 유지했습니다.
- DB 스키마와 RLS는 변경하지 않았습니다.
- 화면, 백업, 계정 삭제 요청, 정적 자원과 서비스워커 캐시 버전을 `4.5.4`로 통일했습니다.
