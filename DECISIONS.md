# 훈노트 주요 결정

## 승인

- 사용자별 카테고리 추가·이름 변경·삭제
- 삭제된 카테고리의 메모를 `미분류`로 이동
- 카테고리를 Supabase에 저장해 기기 간 동기화
- Google 로그인 우선 검토, 카카오 로그인은 안정화 후 검토
- 메모 목록은 좌우 페이지 넘김보다 `더 보기` 방식 우선 검토
- 메인 화면에서 중요 표시 변경 기능 검토

## 보류

- 전화번호 로그인: SMS 비용·남용 방지·운영 부담 때문에 출시 후 재검토
- 메뉴가 열린 상태의 당겨서 새로고침: 제스처·스크롤 충돌 우려
- 카테고리 색상·순서 직접 변경: v4.5 범위에서 제외
- `할 일`·`보관` 데이터 구조 전환: 기존 데이터 호환 설계 후 별도 버전에서 진행

## 데이터 원칙

- 카테고리 삭제는 메모 삭제가 아니다.
- 시스템 분류는 사용자 카테고리와 구분한다.
- DB 변경은 기존 사용자 데이터 이전과 RLS 검수를 포함한다.
- 새 기능보다 비공개 테스트 안정성을 우선한다.
