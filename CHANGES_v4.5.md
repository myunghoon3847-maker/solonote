# 훈노트 v4.5 변경 기록

## 목적

고정 카테고리 구조를 사용자별 클라우드 카테고리로 전환하면서 기존 메모를 안전하게 보존합니다.

## 추가

- 사용자별 `memo_categories` 테이블과 RLS 정책
- 카테고리 추가·이름 변경·삭제 UI
- 카테고리 이름 변경과 메모 이동을 한 트랜잭션으로 처리하는 Supabase 함수
- 삭제된 카테고리의 메모를 `미분류`로 이동
- 기본 카테고리 `업무`, `아이디어`, `일상` 자동 생성
- 기존 메모에서 발견한 일반 카테고리 자동 등록
- JSON 백업·복원 시 카테고리 목록 보존

## 안전장치

- 카테고리 이름 1~20자 제한
- 같은 이름과 시스템 이름 등록 방지
- 카테고리 최소 1개 유지
- 이름 변경·삭제 후 메모를 서버에서 다시 불러와 수정 충돌 방지
- `user_id` 기반 RLS와 계정 삭제 시 `ON DELETE CASCADE`

## 호환 유지

- 기존 `업무`, `아이디어`, `일상`, `할 일`, `보관` 메모 보존
- v4.4 이전 JSON 백업 복원 지원
- 기존 로그인·계정 삭제·휴지통·체크리스트·PWA 구조 유지

## 버전 갱신

- 화면 표시 버전 `v4.5`
- 정적 자원 쿼리 `v=450`
- 서비스워커 캐시 `hoonnote-v4-5-cache`
- 계정 삭제 요청 클라이언트 버전 `4.5`
