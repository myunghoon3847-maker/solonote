# 훈노트 Codex 작업 원칙

이 저장소는 1인 개발자·프리랜서·창작자를 위한 단순한 클라우드 업무·아이디어 노트입니다.

## 필수 원칙

- 사용자 승인 없이 새 기능이나 큰 UI 변경을 추가하지 않는다.
- 한 번에 합의된 한 버전 범위만 구현한다.
- 기존 메모·체크리스트·계정 데이터를 삭제하거나 의미를 바꾸지 않는다.
- 데이터베이스 구조 변경은 SQL, 이전 방법, 되돌리기 위험, 테스트 항목을 함께 제공한다.
- 사용자별 데이터 접근에는 Supabase RLS와 `user_id` 검증을 유지한다.
- Supabase 서비스 역할 키, 비밀번호, 서명키를 클라이언트 코드나 문서에 넣지 않는다.
- 공개 가능한 Supabase URL·publishable key만 클라이언트에서 사용한다.
- 모바일 320px부터 PC까지 반응형 레이아웃을 유지한다.
- 구현 후 JavaScript 문법, 핵심 사용자 흐름, PWA 캐시 버전, 기존 기능 회귀를 검수한다.
- 배포 전 SQL 적용 순서와 사용자의 실기기 검수 항목을 명확히 안내한다.

## 제품 원칙

- 핵심 흐름은 `로그인 → 메모 찾기 → 작성·수정 → 자동 동기화`다.
- 사용자가 자주 쓰지 않는 기능은 메뉴나 접힌 보조 영역에 둔다.
- 버튼과 설명을 불필요하게 늘리지 않는다.
- `중요`, `보관`, `휴지통`, `할 일` 같은 상태·정보 유형과 사용자 카테고리를 구분한다.
- 위험한 작업에는 결과를 먼저 설명하고 확인을 받는다.

- 기준 버전: v4.5.4
- 프런트엔드: HTML, CSS, JavaScript
- 인증·DB: Supabase
- 배포: GitHub Pages, PWA, PWABuilder TWA
- Android 패키지명: `com.hooncompany.hoonnote`
- 최신 테스트 문서: `TEST_CHECKLIST_v4.5.4.md`
