# SoloNote v4.3.2.2 계정 삭제 테스트 체크리스트

## 배포 전
- [ ] 사전 점검 SQL의 orphan_memo_count가 0이다.
- [ ] ON DELETE CASCADE SQL을 적용했다.
- [ ] delete-account Edge Function을 배포했다.
- [ ] 브라우저 코드에 service_role 키가 없다.

## 화면
- [ ] 메뉴에 계정 삭제 버튼이 보인다.
- [ ] 삭제 창에서 경고와 삭제 항목이 명확하다.
- [ ] 비밀번호와 확인 문구가 없으면 삭제 버튼이 비활성화된다.
- [ ] `계정 삭제`를 정확히 입력해야 버튼이 활성화된다.
- [ ] 취소, 배경 클릭, Esc로 창이 닫힌다.
- [ ] 삭제 진행 중에는 창을 닫거나 중복 요청할 수 없다.

## 기능
- [ ] 잘못된 비밀번호는 삭제되지 않고 오류가 표시된다.
- [ ] Edge Function 미배포 시 이해 가능한 오류가 표시된다.
- [ ] 정상 삭제 시 Auth 사용자가 사라진다.
- [ ] 정상 삭제 시 해당 사용자의 memos 행이 사라진다.
- [ ] 정상 삭제 시 로컬 초안과 이전 메모가 정리된다.
- [ ] 삭제 후 로그인 화면에 완료 메시지가 보인다.
- [ ] 삭제한 이메일과 비밀번호로 로그인할 수 없다.

## 회귀 테스트
- [ ] 회원가입과 이메일 인증이 정상이다.
- [ ] 로그인, 로그아웃, 비밀번호 재설정이 정상이다.
- [ ] 메모 작성, 수정, 삭제, 복원이 정상이다.
- [ ] 검색, 필터, 할 일, 백업이 정상이다.
- [ ] PC와 스마트폰 화면이 깨지지 않는다.
