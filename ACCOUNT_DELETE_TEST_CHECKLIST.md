# 훈노트 v4.3.3 계정 삭제 테스트 체크리스트

## A. Supabase 배포 전 확인

- [ ] `01_account_delete_preflight.sql`의 `orphan_memo_count`가 0이다.
- [ ] `02_enable_account_delete_cascade.sql`을 적용했다.
- [ ] `03_create_account_deletion_guard.sql`을 적용했다.
- [ ] `04_account_delete_verify.sql`의 `cascade_enabled`가 true다.
- [ ] `guard_function_installed`가 true다.
- [ ] `delete-account` Edge Function을 v4.3.3 코드로 배포했다.
- [ ] 브라우저 코드와 GitHub에 관리자 키가 없다.

## B. 화면과 입력 검수

- [ ] 우측 메뉴의 계정 관리에서 계정 삭제 버튼이 보인다.
- [ ] 삭제되는 항목과 복구 불가 안내가 명확하다.
- [ ] JSON 백업 버튼이 작동한다.
- [ ] 비밀번호가 비어 있으면 삭제 버튼이 비활성화된다.
- [ ] `계정 삭제`를 정확히 입력해야 삭제 버튼이 활성화된다.
- [ ] 취소·배경 클릭·Esc로 창이 닫힌다.
- [ ] 삭제 진행 중에는 창 닫기와 중복 요청이 차단된다.

## C. 실패 안전성

- [ ] 잘못된 비밀번호 입력 시 계정과 메모가 모두 유지된다.
- [ ] 인터넷을 끊고 시도하면 이해 가능한 네트워크 오류가 표시된다.
- [ ] Edge Function 미배포 상태에서 함수 미배포 오류가 표시된다.
- [ ] 03 SQL 미적용 상태에서는 서버 안전 점검 오류가 표시되고 데이터가 유지된다.
- [ ] CASCADE가 준비되지 않은 상태에서는 삭제가 거부되고 데이터가 유지된다.
- [ ] 세션 만료 상태에서는 재로그인 안내가 표시된다.
- [ ] 삭제 버튼을 여러 번 눌러도 요청이 중복 실행되지 않는다.

## D. 정상 삭제

- [ ] 올바른 비밀번호로 최근 password 인증이 생성된다.
- [ ] Auth 사용자가 삭제된다.
- [ ] 해당 사용자의 `memos` 행이 0개가 된다.
- [ ] 삭제한 계정으로 다시 로그인할 수 없다.
- [ ] localStorage의 `solonote_` 데이터가 제거된다.
- [ ] sessionStorage의 훈노트 임시 데이터가 제거된다.
- [ ] Supabase 로컬 인증 토큰이 제거된다.
- [ ] 로그인 화면에 삭제 완료 메시지가 표시된다.

## E. 다른 사용자 보호

- [ ] 계정 A 삭제 전 계정 B의 메모 개수를 기록했다.
- [ ] 계정 A 삭제 후 계정 B의 메모 개수가 동일하다.
- [ ] 계정 B 로그인·메모 작성·수정이 정상이다.
- [ ] Edge Function 요청에 사용자 ID를 임의로 넣어도 다른 계정을 지정할 수 없다.

## F. 회귀 테스트

- [ ] 회원가입과 이메일 인증이 정상이다.
- [ ] 로그인·로그아웃·비밀번호 재설정이 정상이다.
- [ ] 메모 작성·수정·휴지통 이동·복원이 정상이다.
- [ ] 영구 삭제와 휴지통 비우기가 정상이다.
- [ ] 검색·카테고리·프로젝트·할 일이 정상이다.
- [ ] JSON 백업과 복원이 정상이다.
- [ ] PC와 스마트폰에서 계정 삭제 화면이 깨지지 않는다.
