## 자동 DOM 기능 검수

Supabase 응답을 모의 구성해 다음 사용자 흐름을 실행했습니다.

1. 기본 카테고리 불러오기
2. `고객 업무` 카테고리 추가
3. `고객 관리`로 이름 변경
4. 해당 카테고리로 메모 저장
5. 카테고리 삭제
6. 저장된 메모가 `미분류`로 이동
7. `미분류` 필터 표시
8. JSON 백업 데이터에 카테고리 포함

결과: 통과

## 데이터 안전 검토

- 카테고리 이름 변경·삭제와 연결 메모 갱신을 PostgreSQL 트랜잭션에서 처리
- 사용자별 RLS와 `auth.uid()` 확인
- 계정 삭제 시 `ON DELETE CASCADE`
- 삭제 시 메모 본문을 제거하지 않고 카테고리 값만 `미분류`로 변경
- 변경 후 서버 메모를 다시 불러와 수정 시각 충돌 방지
