# 훈노트 v4.5.5 변경 기록

## 사용자 요청 반영

1. 상단 훈노트 로고 클릭 시 메모 홈으로 이동
2. 휴지통의 `메모로 돌아가기` 버튼 삭제
3. 스마트폰 화면 확대·입력 자동 확대 방지
4. PWA 독립 실행 설정 유지 및 Android 주소 표시 해결 문서 추가
5. 상단 로고를 간결한 `H` 심볼 + 훈노트 워드마크로 변경
6. 메모 작성 화면에 카테고리 `추가·삭제` 진입 버튼 추가

## 동작 세부사항

- 홈 이동 시 검색어, 선택 카테고리와 정렬을 기본 상태로 되돌립니다.
- 작성 중인 미저장 내용은 홈 이동 전에 브라우저 초안으로 저장하고 복구 안내를 표시합니다.
- 카테고리 추가·삭제는 기존 Supabase 카테고리 관리 기능을 그대로 사용합니다.
- 카테고리를 삭제해도 연결된 메모는 삭제되지 않고 기존 정책대로 `미분류`로 이동합니다.
- 모바일 입력 요소는 16px 이상으로 표시하여 iOS 입력 포커스 자동 확대를 줄였습니다.

## 데이터 및 배포 영향

- DB 스키마 변경 없음
- 추가 SQL 없음
- 기존 메모·카테고리·체크리스트·백업 호환 유지
- 서비스워커 캐시: `hoonnote-v4-5-5-cache`
- 정적 파일 캐시 쿼리: `v=455`
