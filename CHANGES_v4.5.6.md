# 훈노트 v4.5.6 변경 기록

## 수정 목적

v4.5.5 배포 후 일부 기기에서 로고가 기본 버튼처럼 표시되고 홈 이동이 작동하지 않거나, 모바일 카테고리 영역이 화면 밖으로 이어져 보이던 문제를 수정합니다.

## 변경 내용

- 로고를 `button`에서 실제 홈 주소를 가진 `a` 링크로 변경
  - JavaScript가 정상일 때는 현재 앱 상태를 정리하고 메모 홈으로 이동
  - JavaScript가 로드되지 않아도 링크 자체로 홈을 다시 열 수 있는 대체 동작 제공
- 브라우저 기본 버튼 모양을 제거하도록 `appearance: none`과 링크 스타일을 명시
- 서비스워커 등록 주소를 잘못 남아 있던 `v=454`에서 `v=456`으로 수정
- CSS·JavaScript·아이콘·매니페스트 캐시 쿼리를 `v=456`으로 통일
- 서비스워커 캐시명을 `hoonnote-v4-5-6-cache`로 변경
- `html`, `body` 및 주요 앱 컨테이너의 가로 넘침 차단
- 모바일 카테고리 탭을 가로 스크롤 방식에서 줄바꿈 방식으로 변경
- 360px 이하 헤더 간격과 클라우드 상태 표시 폭 보정
- 백업·계정 삭제 요청 버전을 `4.5.6`으로 통일

## 데이터 영향

- Supabase 테이블과 RLS 정책 변경 없음
- 기존 메모·카테고리·체크리스트·백업 형식 호환 유지
- 추가 SQL 실행 및 새 AAB 생성은 웹 수정만 배포할 경우 필요 없음
