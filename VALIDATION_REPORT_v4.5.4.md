# 훈노트 v4.5.4 내부 검수 보고서

## 자동 검수 범위

- 전체 JavaScript와 서비스워커 문법 검사
- v4.5.3 보안 회귀 테스트
  - 악성 체크리스트 ID 정규화와 HTML 이스케이프
  - 과대·손상 백업 데이터 거부
  - 메모 조회·레거시 변환의 `user_id` 조건
- v4.5.4 UI 계약 검사
  - 프로젝트·필터·메모 개수 UI 제거
  - 정렬 텍스트 5종과 구분선
  - 데이터 관리 안의 휴지통 버튼
  - 빈 화면 버튼과 프로젝트 문구 제거
  - 본문 입력 높이
- 내부 화면용 브라우저 기록 계층과 `popstate` 처리 존재 검사
- 화면·백업·계정 삭제·캐시 버전 일치
- HTML과 서비스워커의 로컬 정적 파일 존재 검사

## 실행 결과

- JavaScript·서비스워커 문법 검사: 통과
- Node 회귀 테스트: 7개 중 7개 통과
- JSON 문법 검사: 통과
- 로컬 정적 파일 참조 검사: 통과
- 브라우저 자동 화면 검수 스크립트는 포함했으나, 현재 검수 환경에 Chromium 실행 파일이 없어 실행하지 못했습니다.

## 데이터 호환 판단

- `project` DB 열과 기존 값은 삭제하지 않았습니다.
- 프로젝트는 사용자 화면과 검색에서만 제외했습니다.
- 기존 메모를 수정할 때 숨겨진 프로젝트 값은 유지됩니다.
- 새 메모에는 빈 프로젝트 값이 저장됩니다.
- DB 스키마·RLS·Edge Function 배포는 변경하지 않습니다.

## 배포 후 반드시 확인할 항목

- 실제 Android/PWA 뒤로가기
- 스마트폰 320px·360px·일반 PC의 정렬 줄바꿈과 터치 영역
- 실제 Supabase 작성·수정·휴지통·복원·백업
- 계정 삭제 모달의 열기·취소와 뒤로가기
- 설치 앱의 v4.5.4 서비스워커 갱신

따라서 코드 자동 검수는 통과했지만, 스마트폰·PC 실화면과 물리 뒤로가기는 배포 전 실기기 확인이 필요합니다.

운영 `memos` RLS, 계정 A·B 교차 접근, 실제 AAB·서명 검수는 이번 UI 버전과 별개로 남아 있는 보안 확인 항목입니다.
